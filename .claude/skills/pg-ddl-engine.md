# Skill: PostgreSQL DDL Engine — Types, Constraints, and DDL Generation

## When to Use

Apply this skill when working on `ddl_generator.py`, `type_mapper.py`, `schema_builder.py`, or any backend module that generates PostgreSQL DDL. Covers all PG data types, constraints, table types, index types, identifier safety, source-to-PG type mapping, and FK dependency ordering.

## PostgreSQL Data Types (Complete Catalog)

### Numeric Types

| Type | Size | Range / Notes |
|------|------|---------------|
| `smallint` | 2 bytes | -32768 to +32767 |
| `integer` | 4 bytes | -2147483648 to +2147483647 |
| `bigint` | 8 bytes | -9223372036854775808 to +9223372036854775807 |
| `numeric(p,s)` / `decimal(p,s)` | variable | Exact. p=total digits, s=decimal digits. Up to 131072 digits before decimal, 16383 after. |
| `real` | 4 bytes | 6 decimal digits precision |
| `double precision` | 8 bytes | 15 decimal digits precision |
| `money` | 8 bytes | Currency. Locale-dependent formatting. Avoid in new designs — use `numeric(p,2)` instead. |

### Auto-Increment / Identity

| Type | Notes |
|------|-------|
| `smallserial` | Legacy. Auto-incrementing smallint. Creates implicit sequence. |
| `serial` | Legacy. Auto-incrementing integer. Creates implicit sequence. |
| `bigserial` | Legacy. Auto-incrementing bigint. Creates implicit sequence. |
| `GENERATED ALWAYS AS IDENTITY` | **Modern (PG 10+). Preferred.** SQL-standard. Cannot be overridden by INSERT. |
| `GENERATED BY DEFAULT AS IDENTITY` | Modern. Allows manual value override on INSERT. |

**Always prefer IDENTITY over SERIAL in new schemas.**

```sql
-- Modern (preferred)
"id" BIGINT GENERATED ALWAYS AS IDENTITY

-- Legacy (still works, but avoid in new schemas)
"id" SERIAL
```

### Character Types

| Type | Notes |
|------|-------|
| `varchar(n)` / `character varying(n)` | Variable-length, max n chars. |
| `char(n)` / `character(n)` | Fixed-length, blank-padded. Rarely useful. |
| `text` | Unlimited length. Same performance as varchar in PG. |

### Temporal Types

| Type | Notes |
|------|-------|
| `date` | Date only (no time). 4713 BC to 5874897 AD. |
| `time` | Time only (no date). Microsecond precision. |
| `timetz` / `time with time zone` | Time with timezone. Discouraged — use timestamptz. |
| `timestamp` | Date + time without timezone. |
| `timestamptz` / `timestamp with time zone` | **Preferred for most use cases.** Stores UTC, displays in session timezone. |
| `interval` | Time duration. `'1 year 2 months 3 days 4 hours'`. |

### Boolean

| Type | Values |
|------|--------|
| `boolean` | `TRUE`, `FALSE`, `NULL` |

### UUID

| Type | Notes |
|------|-------|
| `uuid` | 128-bit UUID. Use `gen_random_uuid()` for default (PG 13+, no extension needed). |

### Binary

| Type | Notes |
|------|-------|
| `bytea` | Variable-length binary string. Hex or escape format. |

### JSON / Document

| Type | Notes |
|------|-------|
| `json` | Stores exact text. Re-parsed on every access. |
| `jsonb` | **Preferred.** Binary format. Indexable (GIN). Supports containment operators. |
| `xml` | XML document storage. xpath() support. |
| `hstore` | Key-value pairs (text→text). Requires extension. Prefer jsonb for new designs. |

### Network Types

| Type | Notes |
|------|-------|
| `inet` | IPv4 or IPv6 host address with optional subnet. |
| `cidr` | IPv4 or IPv6 network address. |
| `macaddr` | MAC address (6 bytes). |
| `macaddr8` | MAC address (8 bytes, EUI-64). |

### Geometric Types

| Type | Notes |
|------|-------|
| `point` | (x,y) coordinate. |
| `line` | Infinite line. |
| `lseg` | Line segment. |
| `box` | Rectangular box. |
| `path` | Open or closed path. |
| `polygon` | Closed polygon. |
| `circle` | Circle with center and radius. |

### Text Search Types

| Type | Notes |
|------|-------|
| `tsvector` | Sorted list of distinct lexemes (for full-text search). |
| `tsquery` | Text search query. |

### Range Types

| Type | Notes |
|------|-------|
| `int4range` | Range of integer. |
| `int8range` | Range of bigint. |
| `numrange` | Range of numeric. |

### Bit String Types

## Constraint Types

### PRIMARY KEY

```sql
-- Column-level (single column)
"id" BIGINT PRIMARY KEY

-- Table-level (single or composite)
CONSTRAINT "users_pkey" PRIMARY KEY ("id")
CONSTRAINT "order_items_pkey" PRIMARY KEY ("order_id", "product_id")
```

### FOREIGN KEY

```sql
-- Full syntax with all options
CONSTRAINT "orders_user_id_fkey"
  FOREIGN KEY ("user_id")
  REFERENCES "users" ("id")
  ON DELETE CASCADE
  ON UPDATE NO ACTION
  DEFERRABLE INITIALLY DEFERRED
  MATCH SIMPLE
```

**ON DELETE / ON UPDATE actions:**

| Action | Behavior |
|--------|----------|
| `NO ACTION` | Default. Raise error if referenced row exists. Checked at end of statement. |
| `RESTRICT` | Raise error immediately. Cannot be deferred. |
| `CASCADE` | Delete/update referencing rows automatically. |
| `SET NULL` | Set FK column(s) to NULL. Columns must be nullable. |
| `SET DEFAULT` | Set FK column(s) to default value. Default must satisfy FK. |

**DEFERRABLE options:**
- `NOT DEFERRABLE` — checked immediately (default)
- `DEFERRABLE INITIALLY IMMEDIATE` — can be deferred per-transaction
- `DEFERRABLE INITIALLY DEFERRED` — deferred by default, checked at COMMIT

### UNIQUE

```sql
-- Single column
CONSTRAINT "users_email_key" UNIQUE ("email")

-- Composite
CONSTRAINT "idx_unique_name_org" UNIQUE ("name", "org_id")

-- PG15+: NULLS NOT DISTINCT (two NULLs are considered equal = not unique)
CONSTRAINT "users_email_key" UNIQUE NULLS NOT DISTINCT ("email")
```

### CHECK

```sql
CONSTRAINT "users_age_check" CHECK ("age" >= 0)
CONSTRAINT "products_price_check" CHECK ("price" > 0 AND "price" < 1000000)
CONSTRAINT "orders_status_check" CHECK ("status" IN ('pending', 'shipped', 'delivered'))

-- NO INHERIT: child (partitioned) tables don't inherit this check
CONSTRAINT "base_check" CHECK ("type" IS NOT NULL) NO INHERIT
```

### NOT NULL

```sql
"email" VARCHAR(255) NOT NULL
```

### DEFAULT

```sql
"created_at" TIMESTAMPTZ NOT NULL DEFAULT NOW()
"is_active" BOOLEAN NOT NULL DEFAULT TRUE
"id" UUID NOT NULL DEFAULT gen_random_uuid()
"status" VARCHAR(20) NOT NULL DEFAULT 'pending'
"count" INTEGER NOT NULL DEFAULT 0
```

### IDENTITY (Modern Auto-Increment)

```sql
"id" BIGINT GENERATED ALWAYS AS IDENTITY
"id" INTEGER GENERATED BY DEFAULT AS IDENTITY
```

## Table Types

### Permanent (default)

```sql
CREATE TABLE "users" ( ... );
```

### Temporary

```sql
CREATE TEMP TABLE "session_data" ( ... );
-- Auto-dropped at end of session. Not visible to other sessions.
-- ON COMMIT options: PRESERVE ROWS (default), DELETE ROWS, DROP
CREATE TEMP TABLE "temp_calc" ( ... ) ON COMMIT DELETE ROWS;
```

### Unlogged

```sql
CREATE UNLOGGED TABLE "event_buffer" ( ... );
-- No WAL logging = much faster writes.
-- DATA LOST ON CRASH. Not replicated to standbys.
-- Good for: caches, ETL staging, ephemeral data.
```

## Index Types

```sql
-- B-tree (default): equality and range queries
CREATE INDEX "users_email_idx" ON "users" ("email");

-- Hash: equality-only (slightly faster than B-tree for =)
CREATE INDEX "users_token_idx" ON "users" USING HASH ("token");

-- GIN: jsonb, arrays, full-text search
CREATE INDEX "products_tags_idx" ON "products" USING GIN ("tags");
CREATE INDEX "docs_search_idx" ON "docs" USING GIN (to_tsvector('english', "content"));

-- GiST: geometric, range types, full-text search
CREATE INDEX "locations_point_idx" ON "locations" USING GIST ("coordinates");

-- BRIN: very large naturally-ordered tables (e.g., time-series)
CREATE INDEX "events_created_idx" ON "events" USING BRIN ("created_at");

-- Partial index: only index rows matching a condition
CREATE INDEX "orders_active_idx" ON "orders" ("status") WHERE "status" = 'active';

-- Unique index
CREATE UNIQUE INDEX "users_email_unique_idx" ON "users" ("email");

-- Expression index
CREATE INDEX "users_lower_email_idx" ON "users" (LOWER("email"));
```

## DDL Generation Patterns

### Full Table DDL

```python
def generate_create_table(table: dict) -> str:
    """Generate CREATE TABLE statement from table definition.

    Args:
        table: {
            name: str,
            tableType: 'permanent' | 'temp' | 'unlogged',
            columns: list[dict],
            constraints: list[dict],
            ifNotExists: bool,
            comment: str | None,
        }
    """
    parts = []

    # Table type prefix
    prefix = {
        "permanent": "CREATE TABLE",
        "temp": "CREATE TEMP TABLE",
        "unlogged": "CREATE UNLOGGED TABLE",
    }[table["tableType"]]

    exists = "IF NOT EXISTS " if table.get("ifNotExists") else ""
    parts.append(f'{prefix} {exists}"{table["name"]}" (')

    # Columns
    col_defs = []
    for col in table["columns"]:
        col_defs.append("  " + generate_column_def(col))

    # Table-level constraints
    for constraint in table.get("constraints", []):
        col_defs.append("  " + generate_constraint_def(constraint))

    parts.append(",\n".join(col_defs))
    parts.append(");")

    sql = "\n".join(parts)

    # Table comment
    if table.get("comment"):
        comment = table["comment"].replace("'", "''")
        sql += f'\n\nCOMMENT ON TABLE "{table["name"]}" IS \'{comment}\';'

    return sql
```

### Column Definition

```python
def generate_column_def(col: dict) -> str:
    """Generate a single column definition line."""
    parts = [f'"{col["name"]}"']

    # Type + identity
    if col.get("identity"):
        base_type = col["type"].upper()
        parts.append(f"{base_type} GENERATED {col['identity']} AS IDENTITY")
    else:
        parts.append(format_type(col["type"]))

    # NOT NULL
    if not col.get("nullable", True):
        parts.append("NOT NULL")

    # DEFAULT
    if col.get("defaultValue") is not None:
        parts.append(f"DEFAULT {col['defaultValue']}")

    return " ".join(parts)
```

### Constraint Definition

```python
def generate_constraint_def(constraint: dict) -> str:
    """Generate a table-level constraint definition."""
    cols = ", ".join(f'"{c}"' for c in constraint["columns"])

    if constraint["type"] == "pk":
        return f'CONSTRAINT "{constraint["name"]}" PRIMARY KEY ({cols})'

    if constraint["type"] == "fk":
        ref_cols = ", ".join(f'"{c}"' for c in constraint["refColumns"])
        parts = [
            f'CONSTRAINT "{constraint["name"]}"',
            f'FOREIGN KEY ({cols})',
            f'REFERENCES "{constraint["refTable"]}" ({ref_cols})',
        ]
        if constraint.get("onDelete", "NO ACTION") != "NO ACTION":
            parts.append(f'ON DELETE {constraint["onDelete"]}')
        if constraint.get("onUpdate", "NO ACTION") != "NO ACTION":
            parts.append(f'ON UPDATE {constraint["onUpdate"]}')
        return " ".join(parts)

    if constraint["type"] == "unique":
        return f'CONSTRAINT "{constraint["name"]}" UNIQUE ({cols})'

    if constraint["type"] == "check":
        return f'CONSTRAINT "{constraint["name"]}" CHECK ({constraint["expression"]})'

    return ""
```

## FK Dependency Ordering (Topological Sort)

Tables must be created in dependency order. Use Kahn's algorithm:

```python
def topological_sort_tables(tables: list[dict]) -> tuple[list[dict], list[dict]]:
    """Sort tables by FK dependencies. Returns (ordered_tables, circular_fk_constraints).

    Tables with no FK dependencies come first.
    If circular FKs exist, those constraints are separated for ALTER TABLE pass.
    """
    # Build adjacency: table_name -> set of tables it depends on (FK targets)
    deps: dict[str, set[str]] = {t["name"]: set() for t in tables}
    all_fk_constraints: dict[str, list[dict]] = {t["name"]: [] for t in tables}

    for table in tables:
        for constraint in table.get("constraints", []):
            if constraint["type"] == "fk":
                ref = constraint["refTable"]
                if ref != table["name"]:  # skip self-refs
                    deps[table["name"]].add(ref)
                all_fk_constraints[table["name"]].append(constraint)

    # Kahn's algorithm
    in_degree = {name: len(d) for name, d in deps.items()}
    queue = [name for name, deg in in_degree.items() if deg == 0]
    ordered_names: list[str] = []

    while queue:
        node = queue.pop(0)
        ordered_names.append(node)
        for name, d in deps.items():
            if node in d:
                d.remove(node)
                in_degree[name] -= 1
                if in_degree[name] == 0:
                    queue.append(name)

    # Detect circular dependencies
    circular_fks: list[dict] = []
    remaining = [name for name in deps if name not in ordered_names]

    if remaining:
        # Break cycles: move FK constraints to ALTER TABLE pass
        for name in remaining:
            ordered_names.append(name)
            for fk in all_fk_constraints[name]:
                if fk["refTable"] in remaining:
                    circular_fks.append({"table": name, "constraint": fk})

    # Map names back to table dicts
    table_map = {t["name"]: t for t in tables}
    ordered = [table_map[name] for name in ordered_names if name in table_map]

    return ordered, circular_fks
```

## Full DDL Export Assembly

```python
def generate_full_ddl(schema: dict) -> str:
    """Generate complete PostgreSQL DDL for a schema.

    Order:
    1. Header comment + encoding
    2. BEGIN transaction
    3. CREATE TYPE (enums)
    4. CREATE TABLE (topologically sorted)
    5. ALTER TABLE (circular FK constraints)
    6. CREATE INDEX
    7. COMMENT ON (table and column comments)
    8. COMMIT
    """
    lines = [
        f"-- Generated by DB Diagram Visualizer",
        f"-- Schema: {schema.get('name', 'public')}",
        f"SET client_encoding = 'UTF8';",
        f"",
        f"BEGIN;",
        f"",
    ]

    # Enums
    for enum in schema.get("enums", []):
        values = ", ".join(f"'{v}'" for v in enum["values"])
        lines.append(f'CREATE TYPE "{enum["name"]}" AS ENUM ({values});')
    if schema.get("enums"):
        lines.append("")

    # Tables (sorted)
    tables = schema.get("tables", [])
    ordered, circular_fks = topological_sort_tables(tables)

    for table in ordered:
        # Remove circular FK constraints (added via ALTER TABLE later)
        clean_table = remove_circular_fks(table, circular_fks)
        lines.append(generate_create_table(clean_table))
        lines.append("")

    # Circular FK constraints via ALTER TABLE
    if circular_fks:
        lines.append("-- Circular FK constraints (added after all tables exist)")
        for item in circular_fks:
            table_name = item["table"]
            fk = item["constraint"]
            cols = ", ".join(f'"{c}"' for c in fk["columns"])
            ref_cols = ", ".join(f'"{c}"' for c in fk["refColumns"])
            line = (
                f'ALTER TABLE "{table_name}" '
                f'ADD CONSTRAINT "{fk["name"]}" '
                f'FOREIGN KEY ({cols}) '
                f'REFERENCES "{fk["refTable"]}" ({ref_cols})'
            )
            if fk.get("onDelete", "NO ACTION") != "NO ACTION":
                line += f' ON DELETE {fk["onDelete"]}'
            if fk.get("onUpdate", "NO ACTION") != "NO ACTION":
                line += f' ON UPDATE {fk["onUpdate"]}'
            lines.append(line + ";")
        lines.append("")

    # Indexes
    for table in ordered:
        for idx in table.get("indexes", []):
            lines.append(generate_index(table["name"], idx))
    if any(t.get("indexes") for t in ordered):
        lines.append("")

    lines.append("COMMIT;")

    return "\n".join(lines)
```

## Identifier Quoting

Always quote all identifiers. The `quote_identifier()` function lives in `ddl_generator.py`. All identifier **validation** (reserved words, injection patterns, naming rules) lives in `pg_validator.py` — see `pg-validation.md` for the canonical implementation.

```python
def quote_identifier(name: str) -> str:
    """Quote a PostgreSQL identifier safely. Lives in ddl_generator.py."""
    if '"' in name:
        raise ValueError(f"Cannot quote identifier containing double quote: {name}")
    return f'"{name}"'
```

The reserved words set (`PG_RESERVED_WORDS`) and injection pattern regex live in `pg_validator.py` as the **single source of truth**. `ddl_generator.py` imports from `pg_validator` when needed.

### Common Reserved Words (Reference)

These are the PostgreSQL reserved words most likely to appear as table/column names. All are auto-quoted, but worth knowing during DDL generation:

| Likely as table name | Likely as column name |
|---------------------|-----------------------|
| `user`, `order`, `group`, `table` | `column`, `key`, `index`, `check` |
| `select`, `update`, `insert`, `delete` | `default`, `constraint`, `primary`, `references` |
| `union`, `join`, `limit`, `offset` | `null`, `not`, `unique`, `foreign` |
| `values`, `from`, `where`, `having` | `time`, `timestamp`, `interval`, `boolean` |

Full list (700+ words) is in `pg_validator.py` `PG_RESERVED_WORDS` set.
